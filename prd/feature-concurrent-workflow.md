# PRD: 并行工作流程设计

## 功能概述

优化图文排版引擎的工作流程，实现**内容编辑**、**样式选择**和**图片上传**三者并行处理，互不干涉，提升用户体验和操作效率。

## 背景

当前工作流是顺序的：
1. Step1: 上传/输入文本
2. Step2: 编辑内容 → 选择样式 → 等待图片上传
3. Step3: 生成预览

**痛点**：
- 图片上传与编辑串行，用户必须等待上传完成才能进行下一步
- 样式选择被阻塞，无法在上传时预览效果
- 操作效率低，特别是在网络慢或图片多的情况下

## 需求设计

### 核心目标
实现**三线程并行工作**：
1. **文本编辑线程**：用户随时增删改内容块
2. **样式选择线程**：用户随时选择/切换装饰样式
3. **图片上传线程**：后台静默上传图片，不阻塞用户操作

### 功能特性

#### 1. Step1: 压缩包上传（不变）
- 支持 zip/7z 拖拽上传
- 解压后并行处理：
  - 文档解析（mammoth）→ 填充文本
  - 图片提取 → **立即开始上传到微信**
- 上传完成后进入 Step2

#### 2. Step2: 并行处理界面

**UI 布局：**
```
┌─────────────────────────────────────────────────┐
│  顶部：图片上传进度条（若未完成）                │
│  ┌─────────────┬─────────────────────────────┐  │
│  │  左侧：内容块编辑  │  右侧：样式选择面板          │  │
│  │                   │                            │  │
│  │  - 添加块         │  - 标题样式选择            │  │
│  │  - 删除块         │  - 正文样式选择            │  │
│  │  - 修改文本       │  - 引言样式选择            │  │
│  │  - 调整顺序       │  - 实时预览效果            │  │
│  └─────────────┴─────────────────────────────┘  │
│                                                │
│  底部：【下一步】按钮（图片上传完成才可点击）    │
└─────────────────────────────────────────────────┘
```

**错误处理：**
- 图片上传失败 → 显示警告但不阻塞编辑
- 所有图片上传失败 → 提示用户"是否继续纯文本模式？"

#### 3. Step3: 图片替换（不变）
- 左右分栏布局
- 左栏：已上传的微信图片（media_id + url）
- 右栏：HTML 预览
- 点击替换逻辑

## 技术方案

### 状态管理

**Pinia Store (appStore.ts)**
```typescript
interface ImageUploadStatus {
  total: number;        // 总图片数
  completed: number;    // 已完成
  failed: number;       // 失败数
  status: 'idle' | 'uploading' | 'completed' | 'failed';
  errors: Array<{fileName: string, error: string}>;
}

state: {
  uploadStatus: Ref<ImageUploadStatus>;
  wechatImages: Ref<WechatImage[]>;
}

actions: {
  startImageUpload(images: File[]): void;  // 开始上传
  updateUploadProgress(progress: ImageUploadStatus): void;
  onImageUploadComplete(image: WechatImage): void;
  onImageUploadFailed(fileName: string, error: string): void;
}
```

### 组件改造

#### 1. Step1TextInput.vue
- 解压后立即调用 `appStore.startImageUpload(files)`
- 不等待上传完成，立即跳转到 Step2

#### 2. Step2Curtain.vue（主要改造）
- **新增顶部区域**：图片上传进度条
- **左侧内容块编辑区**：保持不变
- **右侧样式选择区**：保持不变
- **底部按钮**：上传完成前禁用

```vue
<template>
  <div>
    <!-- 上传进度提示 -->
    <UploadProgress v-if="showProgress" :status="uploadStatus" />

    <!-- 主要编辑区 -->
    <div class="flex gap-4">
      <ContentEditor :blocks="contentBlocks" />
      <StyleSelector />
    </div>

    <!-- 下一步按钮 -->
    <button :disabled="!canProceed">
      下一步
    </button>
  </div>
</template>
```

#### 3. UploadProgress.vue（新增）
- 显示上传进度（3/5）
- 显示每个图片状态（⏳上传中 / ✓完成 / ✗失败）
- 提供"继续纯文本模式"按钮（全部失败时）

### 并行处理机制

**核心原则：图片上传与UI操作完全解耦**

1. **上传线程**：
   - 在 Step1 启动，后台持续运行
   - 使用 Promise.allSettled 并行上传
   - 更新 Pinia 状态
   - 不阻塞任何UI渲染

2. **编辑线程**：
   - 用户可随时操作内容块
   - 可随时切换样式
   - 实时预览装饰效果（使用占位符）

3. **同步点**：
   - 只有【下一步】按钮受上传状态控制
   - 其他所有操作完全并行

### API 接口

**前端 → 后端**
```typescript
POST /api/upload-images
{
  images: File[]
}

Response:
{
  success: boolean;
  data: Array<{
    media_id: string;
    url: string;
    originalName: string;
    status: 'success' | 'failed';
    error?: string;
  }>;
}
```

**实现要点：**
- 后端支持并发上传
- 返回每个图片的上传结果
- 前端根据结果更新状态

## 用户体验设计

### 状态可视化

1. **进度条**：
   - 绿色：已上传
   - 黄色：上传中
   - 灰色：待上传

2. **图片列表**：
   - ✓ image1.jpg
   - ⏳ image2.jpg
   - ✗ image3.jpg（点击查看错误）

3. **整体状态**：
   - 全部完成：显示"✓ 全部上传完成"
   - 部分完成：显示"⚠ 3/5 完成，2个失败"
   - 全部失败：显示"✗ 图片上传失败，是否继续？" + "继续纯文本"按钮

### 错误恢复

**单个图片上传失败：**
- 不影响其他图片的上传
- 在列表中显示失败状态
- 用户可选择：
  - 重试上传
  - 忽略（继续纯文本模式）
  - 稍后在Step3手动替换

**所有图片上传失败：**
- 显示对话框
- 选项1："重试全部上传"
- 选项2："继续纯文本模式"（移除所有图片占位符）

## 边界情况处理

1. **图片数量多（>20张）**：
   - 自动分组上传（5张/批）
   - 避免触发微信API频率限制
   - 显示"上传中，还剩15张..."

2. **网络中断**：
   - 记录上传进度到 localStorage
   - 刷新页面后可继续上传
   - 显示"检测到未完成的图片上传"

3. **快速切换样式**：
   - 上传线程不受样式选择影响
   - 样式预览立即生效（使用占位符）

## 性能优化

1. **上传队列管理**：
   ```typescript
   // 限制同时上传数量
   const CONCURRENT_UPLOADS = 3;
   ```

2. **节流更新**：
   - 进度条更新节流（200ms）
   - 避免频繁触发重渲染

3. **虚拟滚动**：
   - 图片列表使用虚拟滚动（>50张）
   - 减少DOM节点数量

## 风险与应对

| 风险 | 影响 | 应对策略 |
|-----|-----|---------|
| 上传速度慢，用户等待 | 用户体验差 | 后台上传 + 并行编辑 |
| 微信API频率限制 | 上传失败 | 分批上传 + 指数退避 |
| 用户刷新页面 | 上传中断 | localStorage 恢复进度 |
| Step3 图片未全部上传 | 无法替换 | 显示"图片上传中，请稍候" |
