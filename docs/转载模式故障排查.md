# 转载模式功能 - 问题排查指南

## ⚠️ 当前问题

后端服务报 500 错误，原因是：
1. **数据库连接失败** - PostgreSQL 在端口 5433 无法连接
2. **JWT 认证依赖数据库** - 无法验证用户身份

## 🔧 解决方案

### 方案 1：启动 PostgreSQL 数据库（推荐）

1. 检查 PostgreSQL 是否在运行：
```powershell
# Windows检查服务
Get-Service -Name postgresql*
```

2. 如果未运行，启动 PostgreSQL：
```powershell
# 启动 PostgreSQL 服务
Start-Service postgresql-x64-15  # 或对应的版本号
```

3. 检查 `.env` 文件中的数据库配置：
```env
DATABASE_HOST=localhost
DATABASE_PORT=5433
DATABASE_USERNAME=postgres
DATABASE_PASSWORD=你的密码
DATABASE_NAME=layout_engine
```

4. 重启后端服务：
```powershell
cd content-backend
npm run start:dev
```

### 方案 2：使用测试端点（临时方案）

我已经添加了一个无需认证的测试端点：

**端点：** `POST /api/wechat/fetch-article-test`

**前端修改：**

修改 `src/utils/api.ts`：
```typescript
export const wechatApi = {
  // 临时使用测试端点
  fetchArticle: (url: string) => {
    return api.post('/wechat/fetch-article-test', { url })
  },
}
```

这样就不需要 JWT 认证了！

### 方案 3：临时移除 JWT 守卫

修改 `content-backend/src/wechat/wechat.controller.ts`，注释掉 JWT 守卫：

```typescript
@Post('fetch-article')
// @UseGuards(JwtAuthGuard)  // 临时注释
@HttpCode(HttpStatus.OK)
async fetchArticle(@Body() body: { url: string }) {
  // ...
}
```

  -H "Content-Type: application/json" \

### 3. 查看错误日志

后端会在控制台输出详细的错误信息：
- 抓取开始
- HTML 长度
- 提取的标题、作者
- 任何错误信息

## 📋 常见错误

### 错误 1: 数据库连接失败
```
ECONNREFUSED ::1:5433
```

**解决：** 启动 PostgreSQL 服务

### 错误 2: 未能提取到文章内容
```
未能提取到文章内容，请检查链接是否正确
```

**原因：**
1. 链接已过期或被删除
2. 微信更改了 HTML 结构
3. 链接不是微信公众号文章

**解决：**
- 确认链接有效且可在浏览器中打开
- 使用最新的微信文章链接测试

### 错误 3: 网络超时
```
timeout of 30000ms exceeded
```

**解决：**
- 检查网络连接
- 增加超时时间（在 service 中修改 timeout 参数）
