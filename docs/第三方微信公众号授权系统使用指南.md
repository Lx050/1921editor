# 第三方微信公众号授权系统使用指南

## 🎯 系统概述

这个系统实现了第三方微信公众号授权功能，允许用户授权你的平台管理他们的微信公众号，类似135编辑器、微盟等平台的授权机制。

## 📋 功能特性

- ✅ **第三方授权** - 获得用户公众号的完整管理权限
- ✅ **永久授权** - 无30天限制，永久有效（除非用户主动取消）
- ✅ **自动刷新** - 自动管理 access_token，无需用户干预
- ✅ **批量管理** - 可同时管理多个授权的公众号
- ✅ **完整API** - 支持图片上传、图文创建、消息发布等功能

## 🔧 准备工作

### 1. 申请微信开放平台

1. 访问 [微信开放平台](https://open.weixin.qq.com/)
2. 完成企业认证（需要营业执照，300元/年）
3. 创建"第三方平台"应用
4. 获取以下信息：
   - `component_appid` - 第三方平台AppID
   - `component_appsecret` - 第三方平台AppSecret
   - `component_token` - 消息校验Token
   - `component_aes_key` - 消息加密密钥

### 2. 配置授权事件

在微信开放平台后台配置：
- **授权事件接收URL**: `https://your-domain.com/api/wechat/auth/event`
- **公众号消息与事件接收URL**: `https://your-domain.com/api/wechat/callback`

### 3. 配置后端环境变量

在 `content-backend/.env` 中添加：

```bash
# 微信开放平台配置
WECHAT_OPEN_APP_ID=your_component_appid
WECHAT_OPEN_APP_SECRET=your_component_appsecret
WECHAT_OPEN_TOKEN=your_component_token
WECHAT_OPEN_AES_KEY=your_component_aes_key

# 微信公众号服务端配置（用于自有公众号）
WECHAT_APP_ID=your_wechat_app_id
WECHAT_APP_SECRET=your_wechat_app_secret
```

## 🚀 快速开始

### 1. 前端集成

在需要使用授权功能的页面中引入组件：

```vue
<template>
  <div>
    <WechatAuthManager />
  </div>
</template>

<script setup>
import WechatAuthManager from '@/components/WechatAuthManager.vue';
</script>
```

### 2. 授权流程

1. **添加授权** - 点击"添加新公众号授权"按钮
2. **生成链接** - 系统自动生成授权URL
3. **用户授权** - 公众号管理员扫码或点击链接完成授权
4. **保存授权** - 系统自动保存授权信息到本地和后端

### 3. 使用授权

获得授权后，可以调用各种微信API：

```javascript
import { wechatAuth } from '@/services/ThirdPartyWechatAuth';

// 上传图片
const imageResult = await wechatService.uploadImage(file);

// 创建图文
const draftResult = await wechatService.createDraft({
  app_id: authorizedAppId,
  articles: [{
    title: "文章标题",
    content: "文章内容",
    author: "作者"
  }]
});

// 发布图文
const publishResult = await wechatAuth.publishToWechat(
  authorizedAppId,
  draftResult.media_id
);
```

## 📚 API 参考

### 主要服务类

#### ThirdPartyWechatAuth

主要的授权管理服务类，提供以下方法：

```typescript
// 获取服务实例
const wechatAuth = ThirdPartyWechatAuth.getInstance();

// 初始化存储回调
wechatAuth.initialize({
  storeAuth: (authData) => { /* 存储授权信息 */ },
  getAuth: (appId) => { /* 获取授权信息 */ }
});

// 开始授权流程
const authUrl = await wechatAuth.startAuthorization();

// 处理授权回调
const authInfo = await wechatAuth.handleAuthCallback(code, state);

// 发布内容到公众号
const result = await wechatAuth.publishToWechat(appId, content);

// 获取已授权账号列表
const accounts = await wechatAuth.getAuthorizedAccounts();

// 刷新Token
const refreshedInfo = await wechatAuth.refreshToken(authInfo);

// 移除授权
await wechatAuth.removeAuth(appId);
```

#### wechatService

微信API调用服务：

```typescript
import { wechatService } from '@/services/wechatService';

// 上传图片
const result = await wechatService.uploadImage(file);

// 创建图文草稿
const result = await wechatService.createDraft(articleData);

// 批量上传图片
const results = await wechatService.batchUploadImages(files);
```

### 组件接口

#### WechatAuthManager Props

无props参数，组件完全自包含。

#### 事件监听

组件通过 CustomEvent 触发事件：

```javascript
// 监听授权成功
document.addEventListener('wechat-auth-success', (event) => {
  console.log('授权成功:', event.detail);
});

// 监听授权失败
document.addEventListener('wechat-auth-error', (event) => {
  console.error('授权失败:', event.detail.error);
});
```

## 🔒 安全注意事项

### 1. 访问令牌管理

- 所有 access_token 存储在 localStorage，仅在必要时发送到后端
- 自动检查 token 过期时间，提前5分钟刷新
- 使用 HTTPS 传输所有敏感数据

### 2. 授权验证

- 每次API调用都验证授权状态
- 使用 state 参数防止CSRF攻击
- 验证授权回调的完整性

### 3. 数据保护

- 不在本地存储敏感信息
- 定期清理过期的授权数据
- 使用微信要求的加密传输

## 🛠 故障排查

### 常见问题

1. **授权链接生成失败**
   - 检查 component_appid 是否正确
   - 确认后端服务正常启动
   - 验证网络连接

2. **授权回调处理失败**
   - 检查授权事件接收URL配置
   - 确认 component_token 和 component_aes_key 正确
   - 查看后端日志

3. **API调用失败**
   - 检查 access_token 是否过期
   - 确认公众号权限范围
   - 验证请求参数格式

4. **Token刷新失败**
   - 检查 refresh_token 是否有效
   - 确认授权未被用户取消
   - 查看微信返回的错误码

### 调试方法

1. **查看控制台日志**
   ```javascript
   // 开启调试模式
   console.log('微信授权调试信息:', wechatAuth);
   ```

2. **检查本地存储**
   ```javascript
   // 查看已保存的授权信息
   console.log(localStorage.getItem('wechat_auths'));
   ```

3. **测试API连接**
   ```bash
   # 测试后端服务
   curl http://localhost:3001/api/wechat/test-connection
   ```
