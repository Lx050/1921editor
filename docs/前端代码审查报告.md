## 🚨 严重问题 (必须修复)

### 1. TypeScript 类型错误过多

**位置**: 多个文件

**问题描述**:
项目中有大量的 TypeScript 类型错误，包括：
- 缺少 Vue 组合式 API 的导入
- 使用隐式 any 类型
- 类型定义不匹配

**影响**:
- 代码可维护性降低
- 运行时类型错误风险
- IDE 支持不完整

**修复方案**:
```typescript
// ❌ 问题代码
export function getData() {
  const data = ref(null); // 类型丢失
  return { data };
}

// ✅ 修复后
import { ref, computed } from 'vue';
import type { Ref } from 'vue';

interface UserData {
  id: number;
  name: string;
}

export function getData() {
  const data: Ref<UserData | null> = ref(null);
  return { data };
}
```

### 2. 缺少 vite-env.d.ts 声明文件

**位置**: 项目根目录

**问题描述**:
缺少 Vite 环境类型声明，导致 `import.meta.env` 无法正确识别。

**修复方案**:
```typescript
// 创建 vite-env.d.ts
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_API_BASE_URL: string
  readonly VITE_NODE_ENV: string
  // ...其他环境变量
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}
```

### 3. 虚拟滚动组件缺少必要导入

**位置**: `src/components/VirtualScroll.vue`

**问题描述**:
组件中使用了 `ref`, `computed`, `watch` 但未从 Vue 导入。

**修复方案**:
```vue
<script setup lang="ts">
import { ref, computed, watch, onMounted, onUnmounted } from 'vue';
// 其他代码...
</script>
```

### 4. 微信授权组件类型问题

**位置**: `src/components/WechatAuthManager.vue:473`

**问题描述**:
`lastUsed` 属性类型不匹配（可选 vs 必需）。

**修复方案**:
```typescript
interface WechatAuthInfo {
  authorizer_appid: string;
  authorizer_access_token: string;
  authorizer_refresh_token: string;
  expires_in: number;
  nick_name: string;
  head_img: string;
  func_info: any;
  authorizedAt: number;
  lastUsed?: number; // 改为可选
}
```

### 5. 内存泄漏风险

**位置**: `src/components/VirtualScroll.vue`

**问题描述**:
事件监听器和 ResizeObserver 未在组件卸载时清理。

**修复方案**:
```typescript
onUnmounted(() => {
  if (resizeObserver) {
    resizeObserver.disconnect();
  }
  // 清理其他事件监听器
});
```

### 3. 缺少错误边界处理

**位置**: 多个 API 调用处

**问题描述**:
API 调用缺少统一的错误处理。

**建议**:
```typescript
// 创建 utils/errorHandler.ts
export const handleApiError = (error: unknown) => {
  console.error('API Error:', error);
  if (error instanceof Error) {
    return error.message;
  }
  return '未知错误';
};
```

### 4. 性能问题

**问题描述**:
- 大文件 vendor-docx.js (488KB) 未使用动态导入
- 缺少组件懒加载
- 列表未使用虚拟滚动（虽然有组件但未使用）

**优化方案**:
```typescript
// 动态导入大文件
const loadDocx = () => import('docx-preview');

// 路由懒加载
const routes = [
  {
    path: '/heavy-component',
    component: () => import('./HeavyComponent.vue')
  }
];
```

### 5. 环境变量处理不当

**位置**: 多处使用 `process.env`

**问题描述**:
Vite 项目应使用 `import.meta.env` 而非 `process.env`。

**修复方案**:
```typescript
// ❌ 错误
const API_URL = process.env.REACT_APP_API_URL;

// ✅ 正确
const API_URL = import.meta.env.VITE_API_BASE_URL;
```
