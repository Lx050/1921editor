# 微信授权续期原理详解

## 🔄 微信OAuth授权流程

### 第一次授权（用户手动操作）
```
用户点击微信登录
    ↓
跳转到微信授权页面
    ↓
用户确认授权
    ↓
微信回调，返回 code
    ↓
用 code 换取 access_token + refresh_token
    ↓
保存 refresh_token（有效期30天）
```

### 授权返回的数据
```json
{
  "access_token": "ACCESS_TOKEN",      // 2小时有效，用于调用API
  "expires_in": 7200,                   // access_token有效期（秒）
  "refresh_token": "REFRESH_TOKEN",    // 30天有效，用于刷新access_token
  "openid": "OPENID",                   // 用户唯一标识
  "scope": "snsapi_userinfo",           // 授权范围
  "unionid": "UNIONID"                  // 跨应用用户唯一标识
}
```

## ✅ 续期机制（自动进行）

### refresh_token 刷新流程
```
检查 access_token 是否即将过期（提前5分钟）
    ↓
如果即将过期，使用 refresh_token 刷新
    ↓
POST https://api.weixin.qq.com/sns/oauth2/refresh_token
参数:
- appid: 你的AppID
- grant_type: refresh_token
- refresh_token: 保存的refresh_token
    ↓
获取新的 access_token + 新的 refresh_token
    ↓
更新本地存储的 token
    ↓
用户无感知，继续正常使用
```

## ⏰ 时间线

```
Day 1:  用户首次授权
↓
Day 1-30:  refresh_token 有效期
↓
Day 2:  access_token 过期，自动刷新
↓
Day 4:  access_token 过期，自动刷新
↓
...
↓
Day 29:  access_token 过期，自动刷新
↓
Day 30:  refresh_token 过期 ❌
        需要用户重新授权
```

## 🎯 解决方案核心

### 核心思路
1. **保存 refresh_token** - 这是30天续期的关键
2. **自动检测过期** - 在access_token过期前自动刷新
3. **后台刷新** - 即使用户不使用应用也能刷新
4. **友好提醒** - refresh_token即将过期时提醒用户重新授权

### 实际效果
- 用户30天内只需要授权1次
- 系统自动处理token刷新
- 30天后提醒用户重新授权
- 重新授权后又是一个30天的循环
