
## 二、飞书端配置

### 2.1 创建多维表格
```yaml
步骤:
  1. 登录飞书客户端
  2. 进入"多维表格"
  3. 点击"新建表格"
  4. 选择"创建空白表格"
  5. 命名：版式内容库

配置字段:
  - 内容ID (单行文本)
  - 标题 (单行文本)
  - 编辑者 (单行文本)
  - 图文来源 (单行文本)
  - 文案作者 (单行文本)
  - 分类 (单选)
  - 状态 (单选)
  - 创建时间 (日期)
  - 更新时间 (日期)
  - 图片数量 (数字)
  - 浏览次数 (数字)
  - 内容摘要 (多行文本)
  - 详情链接 (超链接)

设置单选选项:
  分类: [默认, 资讯, 产品, 案例, 教程]
  状态: [草稿, 已发布, 已归档]
```

### 2.2 创建数据同步插件
```yaml
1. 进入飞书开发者后台
2. 创建"企业自建应用"
3. 配置应用信息:
   - 应用名称: 版式内容同步
   - 应用描述: 同步版式装配引擎内容数据

4. 申请权限:
   - 获取与更新多维表格
   - 查看、评论、编辑和管理多维表格
   - 读取和发送消息

5. 获取凭证:
   - App ID
   - App Secret
   - Verification Token (用于Webhook签名)
```

### 2.3 配置连接器
```yaml
1. 在多维表格中:
   - 点击"连接器" → "从其他数据源同步"
   - 选择"自定义同步应用"

2. 配置插件:
   - 应用服务地址: https://your-domain.com/sync-plugin
   - Verification Token: (从飞书获取的token)

3. 测试连接:
   - 确保能够访问到您的服务
   - 检查接口响应正确
```

### 4.4 实现记录接口
```javascript
// records.js
export async function handleRecords(params) {
  const { maxPageSize, pageToken } = params;

  // 从后端获取数据
  const response = await fetch('http://localhost:3001/api/content/index');
  const index = await response.json();

  // 分页处理
  const offset = pageToken ? parseInt(pageToken) : 0;
  const records = index.slice(offset, offset + maxPageSize);

  return {
    code: 0,
    data: {
      hasMore: offset + maxPageSize < index.length,
      nextPageToken: offset + maxPageSize < index.length ? (offset + maxPageSize).toString() : null,
      records: records.map(item => ({
        primaryID: item.id,
        data: {
          id: item.id,
          title: item.title,
          editor: item.editor,
          source: item.source,
          author: item.author,
          category: item.category,
          status: item.status,
          createdAt: new Date(item.createdAt).getTime(),
          updatedAt: new Date(item.updatedAt).getTime(),
          imageCount: item.imageCount,
          viewCount: item.viewCount || 0,
          summary: item.summary,
          detailUrl: `https://your-domain.com/content/${item.id}`
        }
      }))
    }
  };
}
```

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

    # 静态文件服务
    location /sync-plugin/ {
        alias /path/to/content-backend/sync-plugin/;
```

### 6.2 性能监控脚本
```bash
#!/bin/bash
# monitor.sh

# 检查服务状态
if ! pm2 list | grep -q "content-backend.*online"; then
    echo "服务未运行，正在重启..."
    pm2 restart content-backend
fi

# 检查磁盘空间
DISK_USAGE=$(df /path/to/data | tail -1 | awk '{print $5}' | sed 's/%//')
if [ $DISK_USAGE -gt 80 ]; then
    echo "警告：磁盘使用率超过80%"
    # 发送告警
fi

# 检查内存使用
MEM_USAGE=$(pm2 jlist | jq '.[0].monit.memory')
if [ $MEM_USAGE -gt 800000000 ]; then
    echo "警告：内存使用超过800MB"
fi
```

### 6.3 备份脚本
```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/var/backups/content-data"
DATE=$(date +%Y%m%d-%H%M%S)
DATA_DIR="/path/to/content-backend/data"

# 创建备份
tar -czf "$BACKUP_DIR/content-$DATE.tar.gz" -C "$DATA_DIR" .

# 清理旧备份（保留30天）
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

echo "备份完成: content-$DATE.tar.gz"
```
