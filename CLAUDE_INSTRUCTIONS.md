# Claude Code 项目约束说明

## 重要提醒
此文件用于确保 Claude Code 始终遵守项目约束。Claude Code 会自动读取用户主目录下的 `.claude/CLAUDE.md` 文件作为全局指令。

## 核心原则（必须遵守）

### 1. 语言和沟通
- **中文回答所有问题**
- 每次修改文件前必须说明修改原因和遵循的核心原则

### 2. 开发环境约束
- **禁止执行** `npm run dev`（前端）或 `npm run start:dev`（后端）
- 用户始终保持前后端服务运行
- 开发任务结束前，必须终止正在运行的服务

### 3. 代码修改原则
- **最简原则**：不做任何硬功能的冗余设计，仅对必要的多可能事件做冗余
- **不做向后兼容**：删除所有冗余代码，宁愿破坏性更新
- **复用现有模块**：遵循当前代码中已有的设计风格和命名风格

### 4. TypeScript 规范
- **类型严格**：所有 TypeScript 类型必须正确，禁用 `any`
- **编译错误优先**：遇到问题时，优先考虑编译错误而不是缓存问题
- **立即修复**：TypeScript 编译错误必须立即修复

### 5. 任务管理
- **必须使用 TodoWrite 工具**规划和跟踪任务进度
- 完整阅读所有相关文件，一行都不能少
- 修改后检查是否有重复功能，遵循单一职责原则

### 6. 数据库操作
- **必须使用 TypeORM Migration 方法**
- 遵循迁移脚本编写规范（原子化操作、时间戳命名、KISS原则）

### 7. 其他重要约束
- 注释丰富
- 日志输出不用太多，但前后端关键地方都需要输出
- 记得更新项目文档
- 文档置信度原则：绝不基于推测写代码，必须基于真实可验证的技术文档

## 验证方式
Claude Code 会在每次会话开始时自动读取并应用这些约束。确保以下文件存在且内容正确：
- `C:\Users\Lx050\.claude\CLAUDE.md` - 全局约束文件

## 注意事项
- 这些约束覆盖所有默认行为
- Claude Code 必须严格遵守这些指令
- 如需修改约束，请编辑全局 CLAUDE.md 文件
